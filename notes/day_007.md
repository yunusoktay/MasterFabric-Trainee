# Day 007

## Focus

- Understand inheritance in OOP
- Use `extends` to create a subclass
- Use `super` to reach the superclass
- Override methods with `@override`

## Run

From the repository root:

```bash
dart run dart/day_007.dart
```

## Outcome

Day 7 introduces inheritance with a simple `Vehicle` and `Car` example.

Key ideas:

- A `class` can inherit from another class using `extends`
- The parent class is called the **superclass**, the child is the **subclass**
- `super` is used to call the parent constructor or parent methods
- `@override` marks a method that replaces a method from the superclass
- The subclass keeps the parent's properties/methods and can add new ones

In this example:

- `Vehicle` has `brand`, `speed`, `drive()`, and `stop()`
- `Car` extends `Vehicle`, adds `numberOfDoors` and `describeDoors()`
- `Car.drive()` overrides the parent method but still calls `super.drive()` first

## Key Concepts

| Concept | Where in the Code | What It Means |
|---|---|---|
| **Inheritance** | `class Car extends Vehicle` | A class deriving from another class |
| **Superclass** | `Vehicle` | The parent class that is inherited from |
| **Subclass** | `Car` | The child class that inherits |
| **`extends`** | `class Car extends Vehicle` | Keyword that sets up the inheritance link |
| **`super.brand`** | In the constructor | Shortcut to forward a parameter to the parent's property |
| **`super.drive()`** | Inside a method | Calls the parent's method |
| **`@override`** | Above `drive()` | Marks that a method replaces one from the superclass |
| **`this.numberOfDoors`** | In the constructor | Shortcut for the subclass's own property |

### Rule of Thumb

- `super.` → belongs to the **parent**
- `this.` → belongs to **this class**

## Benefits of Inheritance

1. **No code duplication** — `brand` and `speed` are defined only once (in `Vehicle`). Subclasses like `Car`, `Bike`, or `Truck` don't need to redeclare them.
2. **Logical hierarchy** — You can express real-world relationships like "a Car is a Vehicle" directly in code.
3. **Extending behavior** — With `@override` you can either keep the parent's behavior (by calling `super`) or replace it entirely.
