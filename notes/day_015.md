### Challenge 1: Fake API Fetch

Create a function that returns a `Future<Map<String, String>>` and completes after 2 seconds:

```dart
Future<Map<String, String>> fetchUserProfile() async {
  await Future.delayed(const Duration(seconds: 2));
  return {
    'name': 'Yunus Oktay',
    'email': 'yunus@example.com',
  };
}
```

Then call it with `await`:

```dart
final Map<String, String> profile = await fetchUserProfile();
print('name:  ${profile['name']}');
print('email: ${profile['email']}');
```

This demonstrates a classic "request -> wait -> use result" flow.

### Challenge 2: Countdown Stream

Create a stream that emits numbers from 5 down to 1 with a 1-second gap:

```dart
Stream<int> countdownStream(int from) async* {
  for (int i = from; i >= 1; i--) {
    await Future.delayed(const Duration(seconds: 1));
    yield i;
  }
}
```

Consume it with `await for`:

```dart
await for (final int value in countdownStream(5)) {
  print(value);
}
print('liftoff!');
```

`await for` keeps listening until the stream is fully done.

### Challenge 3: Combine Both

First consume a countdown stream, then fetch the user profile:

```dart
await for (final int value in countdownStream(3)) {
  print('countdown: $value');
}
print('countdown done, fetching profile...');

final Map<String, String> profile = await fetchUserProfile();
print('name:  ${profile['name']}');
print('email: ${profile['email']}');
```

This pattern appears often in real apps: wait for a sequence of events, then trigger the next async action.

## Key Concepts

| Concept | What It Means |
|---|---|
| **`Future<T>`** | A single async value that arrives later |
| **`await`** | Pauses an `async` function until the Future completes |
| **`Stream<T>`** | A sequence of async values over time |
| **`async*` + `yield`** | A function that produces stream events |
| **`await for`** | Consumes stream values until stream completion |
| **Sequencing** | `await` ensures "do this, then that" order |

## Common Mistakes to Avoid

- Forgetting `await` when calling `fetchUserProfile()` (you get a `Future`, not the profile)
- Using `for` instead of `await for` when consuming a stream
- Expecting stream values to arrive instantly (each event is delayed by design)
- Mixing independent and dependent async steps without clear ordering

## Why It Matters

Real Flutter apps constantly mix one-time async operations (`Future`) and continuous async events (`Stream`). Day 15 shows how to coordinate both clearly and safely. This is the foundation for workflows like "wait for progress events, then call API, then update UI."
