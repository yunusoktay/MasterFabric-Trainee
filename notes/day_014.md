## Outcome

Day 14 introduces `Stream`, the asynchronous counterpart of `Iterable`.

### Future vs. Stream

| | `Future<T>` | `Stream<T>` |
|---|---|---|
| Emits | **One** value (or error) | **Many** values over time |
| Analogy | A single promise | A pipe that delivers values as they arrive |
| Awaited with | `await future` | `await for (final v in stream)` |
| Finishes | When the value arrives | When the producer signals "done" (or errors) |

Streams are everywhere in Dart and Flutter: user input events, file reads, WebSocket messages, sensor updates, state management.

### Creating a Stream

Three common ways:

```dart
final Stream<String> a = Stream<String>.fromIterable(['a', 'b', 'c']);

final Stream<int> b = Stream<int>.periodic(
  const Duration(milliseconds: 100),
  (count) => count + 1,
);

Stream<int> countDownStream(int from) async* {
  for (int i = from; i >= 1; i--) {
    await Future.delayed(const Duration(milliseconds: 100));
    yield i;
  }
}
```

- `Stream.fromIterable` — turns an existing collection into a stream.
- `Stream.periodic` — emits a value on every tick of a `Duration`, forever (until cancelled).
- `async*` + `yield` — an asynchronous generator. Each `yield` pushes a value to the stream; the function pauses until the consumer is ready for more.

### Listening to a Stream

Two equivalent styles:

```dart
stream.listen((value) => print(value));
```

```dart
await for (final value in stream) {
  print(value);
}
```

- `.listen(...)` — callback-style, returns a `StreamSubscription` immediately.
- `await for` — only usable inside `async` functions. Pauses the function until the stream is done.

### StreamSubscription

`listen()` returns a `StreamSubscription` that lets you control the flow:

```dart
final subscription = stream.listen(onData);

subscription.pause();   // stop receiving events, buffer them
subscription.resume();  // start receiving again
await subscription.cancel(); // stop forever, release resources
```

**Important:** a `Stream.periodic` subscription will keep the program alive indefinitely — always `cancel()` when you are done, otherwise the app never exits (and in Flutter you leak memory).

### listen() callbacks

`.listen` takes up to four arguments:

```dart
stream.listen(
  (value) => print('data: $value'),
  onError: (Object error) => print('error: $error'),
  onDone: () => print('done'),
  cancelOnError: false,
);
```

- **`onData`** — called for each value emitted.
- **`onError`** — called when the stream emits an error. By default the stream keeps going after an error.
- **`onDone`** — called once, when the stream closes normally.
- **`cancelOnError`** — if `true`, the subscription is cancelled on the first error (no `onDone`).

## Key Concepts

| Concept | What It Means |
|---|---|
| **`Stream<T>`** | An asynchronous sequence of `T` values |
| **`async*`** | Marks a function that returns a `Stream` |
| **`yield`** | Emits one value from an `async*` function |
| **`listen()`** | Subscribes to a stream, returns a `StreamSubscription` |
| **`await for`** | Consumes a stream inside an `async` function |
| **`StreamSubscription`** | Handle to the subscription; supports `pause`, `resume`, `cancel` |

## Future/Stream Equivalents

| `Future` world | `Stream` world |
|---|---|
| `Future<T>` → one value | `Stream<T>` → many values |
| `await future` | `await for (final v in stream)` |
| `future.then((v) => ...)` | `stream.listen((v) => ...)` |
| `try { await ... } catch (e)` | `stream.listen(..., onError: (e) => ...)` |
| Completes once | Can complete, error, or run forever |

## Why It Matters

Many things in a mobile app are not "one value later", they are "a series of values over time": tap events, scroll positions, location updates, chat messages, BLE notifications. Futures cannot model that — Streams can. Mastering `listen`, the `StreamSubscription` lifecycle and `async*` generators is the gateway to reactive UI patterns (like Flutter's `StreamBuilder` and BLoC).
