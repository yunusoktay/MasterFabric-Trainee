# Day 009

## Focus

- Understand what a mixin is
- Create a mixin using the `mixin` keyword
- Apply mixins with the `with` keyword
- Share behavior between unrelated classes

## Run

From the repository root:

```bash
dart run dart/day_009.dart
```

## Outcome

Day 9 introduces mixins — a way to share code between classes that are **not** in the same inheritance tree.

The example uses:

- `Logger` mixin with a `log()` method
- `Flyable` mixin with a `fly()` method
- `Bird` and `Plane` both use `with Logger, Flyable` even though they are unrelated classes

## Key Concepts

| Concept | Where in the Code | What It Means |
|---|---|---|
| **`mixin`** | `mixin Logger { ... }` | Declares a reusable piece of behavior |
| **`with`** | `class Bird with Logger, Flyable` | Applies mixins to a class |
| **Multiple mixins** | `with Logger, Flyable` | A class can use many mixins at once |

## Mixin vs extends vs implements

- `extends` → inherit from **one** parent, get code + shape
- `implements` → promise to fulfill **many** contracts, write all code yourself
- `with` → plug in **many** reusable behaviors, code is copied in automatically

## When to Use Mixins

Use a mixin when:

- You want to share the same method/property across classes that are not related
- You would otherwise duplicate the same code in many places
- You don't want to force those classes into a shared superclass

Classic examples:

- Logging
- Serialization
- Caching
- Animation helpers (common in Flutter)
